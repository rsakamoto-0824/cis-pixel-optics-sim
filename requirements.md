# 要件仕様書 — CMOSイメージセンサー画素光学シミュレーター（cis-pixel-optics-sim）

## 1. 目的

CMOSイメージセンサーの集光特性を評価するため、画素構造（画素サイズ・On Chip Lens形状など）を
パラメータとして与え、FDTD法（厳密な波動光学計算）で光の伝搬を計算し、
集光効率・入射角依存性・クロストーク・光の収束状態を可視化するローカルアプリを開発する。

## 2. 想定ユーザー

- イメージセンサーの画素設計・プロセス開発に携わる技術者
- 社内PC上でローカル実行する（外部サービスへのデータ送信は行わない）

## 3. 対象とする画素構造

- 照射方式: 裏面照射（BSI）のみ
- 積層構造（上から順）:
  1. 空気
  2. On Chip Lens（OCL）
  3. 平坦化膜
  4. カラーフィルタ
  5. 反射防止膜・埋め込み酸化膜（任意）
  6. Si（フォトダイオード領域）
- OCL共有方式: 以下の3方式に対応する
  - 1画素1レンズ（通常）
  - 2画素共有OCL（2x1配置、位相差検出画素を想定）
  - 4画素共有OCL（2x2配置、Quad配列を想定）
- 画素間分離（DTI）: Si内の画素境界に格子状に配置する。あり／なしを選択でき、幅・深さを調整できる

## 4. 入力パラメータ

| 分類 | パラメータ | 範囲・初期値 |
|---|---|---|
| 画素 | 画素サイズ（ピッチ） | 0.5〜2.5 µm（最低限保証範囲） |
| OCL | 高さ | 0.1〜1.5 µm |
| OCL | 曲率（レンズ形状） | 球面キャップ／スーパー楕円で指定 |
| OCL | 共有方式 | 1画素／2画素共有／4画素共有 |
| OCL | 混在パターン | 共有ありなしのレンズを左から並べて指定（例: 1,2,1）。最大8枚、2Dのみ。指定時は共有方式より優先（2026-07-18追加） |
| OCL | 中心オフセット（偏心） | レンズ半幅未満、初期値0。フットプリント（底面の範囲）は画素に固定したまま頂点位置だけをずらし、非対称形状になる（2026-07-13にユーザー指摘で再定義） |
| OCL | ベース層厚 | 0〜2 µm、初期値0。レンズ底面下の同一樹脂の平坦層（光路長調整）。レンズ最低部〜CF間の厚み = ベース層 + 平坦化膜 |
| OCL | 左右ギャップ高さ | 0〜レンズ高さ未満、初期値0。レンズ端（ギャップ部分）の材料高さを左右個別に指定。左右で変えるとレンズが傾いた状態を評価できる |
| OCL | 屈折率（レンズ材） | 1.2〜2.5、初期値1.58。画面から調整可、スイープ対象。ベース層も同じ屈折率 |
| OCL | 表面反射防止膜 | 厚さ0〜0.3 µm（0=膜なし、初期値0）、屈折率1.0〜2.5（初期値1.26≈√1.58）。レンズ表面を覆う薄膜で、両方ともスイープ対象（2026-07-20追加） |
| 光 | 波長 | 400〜700 nm（単波長またはRGB代表波長） |
| 光 | 入射角（CRA） | 0〜35°（方位角も指定可能） |
| 層構造 | 各層の膜厚 | 平坦化膜・カラーフィルタ・Si深さを個別指定（Si厚の初期値は3.5 µm。2026-07-18変更） |
| 層構造 | 各層の屈折率 | 初期値を内蔵し、ユーザーが上書き可能 |
| DTI | あり／なし | 初期値: あり |
| DTI | 幅 | 0.05〜0.3 µm |
| DTI | 深さ | 0〜Si厚（Si貫通のフルDTIを含む） |
| DTI | 配置 | 全画素境界／共有単位境界のみ（共有OCL時に選択可能） |
| DTI | 充填材屈折率 | 初期値: SiO2（n≈1.46）、上書き可能 |
| DTI | 位置オフセット | ±画素サイズ/2、初期値0。DTI格子のみずらす（OCL偏心と組合せで任意の位置ずれを再現） |
| 計算 | 2D／3Dモード | 2D=断面高速評価、3D=真上ビュー・共有OCL評価 |
| 計算 | メッシュ解像度 | 初期値: 100セル/µm（Si内波長基準、design.md 3.4参照）。精度と時間のトレードオフを表示 |

## 5. 出力（評価項目）

1. **集光効率**: フォトダイオード領域（Si内の指定深さ範囲）へ入るパワー ÷ 入射パワー
2. **入射角依存性（CRA特性）**: 入射角スイープに対する集光効率のグラフ
3. **隣接画素クロストーク**: 隣接画素のフォトダイオード領域へ漏れ込むパワー比率
4. **スポット形状の可視化**:
   - 断面ビュー（XZ断面）の電場強度分布 |E|²
   - Si内部のみを拡大スケールで表示するパネル（Si内の光の広がりを確認）
   - 真上ビュー（XY平面、指定深さのスライス）の電場強度分布 |E|²
5. パラメータスイープ結果の一覧表とグラフ（CSVダウンロード可能）
6. CSV一括計算結果の一覧表（条件名・入力値・集光効率・画素ごとの効率をCSVダウンロード可能）

## 6. 機能要件

- F-01: ブラウザ（ローカルWeb UI）からパラメータを入力し、計算を実行できる
- F-02: 計算はジョブとして実行し、進行状況（実行中／完了／失敗）を画面で確認できる
- F-03: 計算結果（画像・グラフ・数値）を画面に表示し、ファイルとして保存できる
- F-04: パラメータスイープ（画素サイズ、OCL高さ、波長、入射角、OCL曲率、膜厚）を指定できる
- F-05: 構造プレビュー（計算前に断面構造図を表示）で入力ミスを防ぐ
  - 構造図（プレビュー・断面図）では空気を背景色（白）で描き、
    OCLなどの材料だけに色を付けて見分けやすくする（2026-07-18 ユーザー指示）
- F-05b: 入力項目はカテゴリ別のタブ（構造／OCL／DTI／光／評価／スイープ）に
  分け、スクロールせずに入力できるようにする
  （選択中のタブはブラウザに記憶。2026-07-18 追加、タブ名は2026-07-19 変更）
- F-05c: メイン画面は「パラメータ設定」「構造プレビュー」「計算結果」の
  3ウィンドウ構成とする（2026-07-19 ユーザー指示で変更）
- F-06: 過去の計算結果を一覧から再表示できる
  - ジョブ履歴はメイン画面には常時表示せず、ヘルプと同様にヘッダーの
    「ジョブ履歴」ボタンから別ページ（別タブ）で開く
    （ジョブ一覧と過去の結果を表示。2026-07-19 ユーザー指示で変更。
    従来の折りたたみ機能はこの変更で廃止）
  - ジョブには内容が分かる名前を自動で付け、一覧に表示する
    （例: スイープ 入射角 [deg] 1µm画素 550nm）
  - ジョブ名は一覧から編集でき、不要になったジョブは削除できる
    （実行中のジョブは中断してから削除。2026-07-18 ユーザー指示で追加）
  - チェックボックスで選択したジョブの一括削除、全ジョブの一括削除ができる
    （いずれも確認のうえ実行。実行中のジョブはスキップして件数を表示）
- F-07: 2Dモード（分オーダー）と3Dモード（数十分オーダー）を選択できる
  - 画面は2D断面モードを基本とする（2026-07-12 ユーザー指示で「計算モード」欄を除外。
    3Dモード・クロストーク評価の計算機能はAPI経由で利用可能）
  - 「真上ビュー（3D計算）」欄を有効にした場合のみ3Dモードで計算し、
    指定深さの水平面を真上から見た |E|² 分布を断面図に加えて表示する
    （観察深さは空欄=PD面と同じ。2026-07-18 ユーザー指示で追加）
- F-08: 画面から用語解説（ヘルプ）を開ける
- F-09: OCL偏心・DTI位置オフセットを指定でき、スイープ対象にもできる
  （2D・3D両対応。ギャップ高さ含め3Dでは横方向＝断面と同じ向きに反映。
  2026-07-19 に3D対応を追加）
- F-10: 画素ごとの受光強度（受光内訳）を画面から評価できる
  （共有レンズ単位×3列・中央単位のみ照射。各画素の受光割合とクロストークを表示。
  1画素1レンズは3画素、2画素／4画素共有は6画素の内訳。2026-07-12に共有OCL対応）
- F-11: 構造断面図・プレビューでカラーフィルタ層を色付きの帯と凡例で明示する
- F-13: OCLの共有ありなしが混在する条件（例: 共有なし・2画素共有・共有なし）を
  混在パターンとして指定し、1回の2D計算で画素ごとの集光効率を評価できる
  （位相差検出画素などの混在配列を想定。3Dとは併用不可。
  2026-07-18 ユーザー指示で追加）
  - 受光内訳の評価と併用できる。並べたレンズのうち中央のレンズ
    （偶数枚のときは中央右寄り）だけを照射し、照射レンズの画素を「中央」、
    それ以外への漏れをクロストークとして集計する
    （パターンは2枚以上で指定。2026-07-19 ユーザー指示で追加）
- F-12: シミュレーション条件をCSVファイルで一括入力し、結果をCSVで保存できる
  （1行=1条件・最大100条件。CSVに書いた列だけを上書きし、空欄は画面の入力値を使う。
  テンプレートCSVをダウンロード可能。結果CSVはExcel対応のBOM付きUTF-8。
  2026-07-18 ユーザー指示で追加）
- F-14: RGB 3波長一括評価ができる（R・G・Bの代表波長を1回の実行でまとめて
  計算し、色ごとの集光効率・クロストークを表とグラフで比較。
  波長の初期値はR=620/G=550/B=450 nmで変更可能。
  スイープ・CSV一括計算とは併用不可。2026-07-19 ユーザー指示で追加）
  - 計算セルに複数の画素が含まれる場合（共有レンズ・受光内訳など）は、
    色ごとの画素別の集光効率も表に表示する（2026-07-20 ユーザー指示で追加）
- F-16: OCL表面に薄い反射防止膜を付けた状態を計算できる
  （2D・3D両対応。2026-07-20 ユーザー指示で追加）
- F-15: ジョブ履歴から2件のジョブを選んで結果を左右に並べて比較できる
  （単発計算どうしのときは集光効率（合計）の差も表示。
  2026-07-19 ユーザー指示で追加）

## 7. 非機能要件

- N-01: アプリ全体（Web UI・解析・計算エンジン）は Python 3.12 で統一する
  （本プロジェクト限定のPythonバージョンルール例外。ユーザー承認 2026-07-12。課題 I-001）
- N-02: FDTD計算エンジンには Meep（オープンソース）を使用する。Meep は Python 3.12 に対応
  しているため、アプリ本体と計算エンジンを単一のPython 3.12環境で動かす
- N-03: すべての計算はローカルPC内で完結し、外部サービスへ構造データ・結果を送信しない
- N-04: 2Dモードの1条件は目安5分以内、3Dモードの1条件は目安60分以内（Apple Silicon Mac想定）
- N-05: 対応OS: macOS（開発機）。将来の社内公開はIIS前提（課題 I-005）

## 8. 制約・前提

- 幾何光学（光線追跡）は精度不足のため使用しない。FDTDによる厳密計算を優先する
- 材料の屈折率は波長分散を考慮する（出典は課題 I-003 で決定）
- カラーフィルタの吸収の扱いは課題 I-004 で決定する
- GitHubリポジトリはPrivateとする
- APIキー等の秘密情報は使用しない（全処理ローカル）
