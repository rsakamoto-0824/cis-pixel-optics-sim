# 基本設計・詳細設計 — cis-pixel-optics-sim

## 1. 全体アーキテクチャ

```text
[ブラウザ]
   │ http://localhost:8000
   ▼
[アプリ本体]  Python 3.14.5（venv）
   ├─ Web UI（FastAPI + HTML/CSS/JavaScript）
   ├─ ジョブ管理（実行キュー・進行状況・履歴）
   ├─ 結果解析・グラフ生成（matplotlib）
   │
   │ subprocess（JSONファイル受け渡し）
   ▼
[計算エンジン]  Python 3.13（conda/miniforge環境、pymeep）
   └─ FDTDワーカー（構造生成 → FDTD実行 → 結果JSON/NPZ出力）
```

- **2環境構成の理由**: Meep（pymeep）は conda-forge で Python 3.13 までしか提供されていない
  （2026-07-12時点、v1.34.0）。アプリ本体を Python 3.14.5 とするルールを守るため、
  計算エンジンのみ専用環境に隔離し、サブプロセス呼び出しで接続する。
- 環境間の受け渡しは JSON（入力パラメータ）と NPZ/JSON（結果データ）のファイル経由とし、
  バージョン差による直接依存を持たない。

## 3. FDTD計算設計（engine）

### 3.1 計算領域と境界条件

- 単位系: Meepの慣例に従い µm 基準（a = 1 µm）
- 横方向（X, Y）: 周期境界。斜入射時はBloch周期境界で入射角を表現する
  - 1画素レンズ: 1画素分を計算セルとする
  - 2画素共有: 2×1画素、4画素共有: 2×2画素を計算セルとする
  - クロストーク評価時は3×3画素（中央照射）に拡張する
- 縦方向（Z）: 上下にPML（吸収境界）
- 光源: 平面波（ガウシアン時間波形）。CRAは波数ベクトルの傾きで与える

### 3.2 評価量の算出方法

| 評価量 | 方法 |
|---|---|
| 集光効率 | Si内フォトダイオード領域上面のフラックスモニター透過パワー ÷ 入射パワー |
| クロストーク | 隣接画素のフォトダイオード領域へのフラックス ÷ 入射パワー（3×3画素モデル） |
| 断面ビュー | XZ断面のDFTモニターで \|E\|² を取得 |
| 真上ビュー | 指定深さのXY平面DFTモニターで \|E\|² を取得（3Dモードのみ） |
| CRA特性 | 入射角を外側ループでスイープし、集光効率を集計 |

### 3.3 2D／3Dモード

- 2Dモード: XZ断面の2D FDTD。数分で完了。断面の傾向把握・広いスイープ向け
- 3Dモード: フル3D。真上ビュー・共有OCL（2×2）・クロストークの定量評価向け
- メッシュ解像度の初期値: `RESOLUTION_PIXELS_PER_UM = 50`（=20 nm刻み、波長400 nmで λ/20相当）

### 3.4 材料モデル（初期値、課題 I-003 で出典確定）

| 材料 | 屈折率の扱い |
|---|---|
| OCL樹脂 | n ≈ 1.58（分散小、定数近似から開始） |
| 平坦化膜 | n ≈ 1.50 |
| カラーフィルタ | n ≈ 1.55（吸収の扱いは課題 I-004） |
| Si | 波長分散＋吸収（k）をテーブルで保持 |

## 4. アプリ本体設計（app）

### 4.1 画面構成（ローカルWeb UI）

1. **パラメータ入力画面**: 構造パラメータ・光条件・計算モードのフォーム。構造プレビュー表示
2. **ジョブ一覧画面**: 実行中／完了／失敗の状態、経過時間、キャンセル
3. **結果表示画面**: 集光効率・クロストークの数値、CRAグラフ、断面／真上ビュー画像、CSV出力

### 4.2 API（アプリ内部のローカルAPI）

| メソッド | パス | 内容 |
|---|---|---|
| POST | /api/jobs | 計算ジョブ投入（パラメータJSON） |
| GET | /api/jobs | ジョブ一覧・状態取得 |
| GET | /api/jobs/{id}/result | 結果取得（数値・画像パス） |
| POST | /api/preview | 構造断面プレビュー生成 |

### 4.3 ジョブ実行フロー

1. UIからパラメータJSONを受領し、入力チェック（範囲・組み合わせ）を行う
2. `jobs/<job_id>/input.json` を書き出し、Meep環境のPythonで `engine/fdtd_worker.py` を起動
3. ワーカーは進捗を `progress.json` に逐次書き出し、完了時に `result.json` / `fields.npz` を出力
4. アプリ本体が結果を読み込み、matplotlibで画像・グラフを生成して表示する

## 5. エラー処理・入力チェック方針

- パラメータ範囲外（画素サイズ 0.5〜2.5 µm 外など）は実行前にエラー表示する
- OCL高さ・曲率の組み合わせが幾何的に成立しない場合（レンズがはみ出す等）はプレビューで警告する
- ワーカー異常終了時は stderr をジョブログとして保存し、UIに「失敗」と理由を表示する
- 3Dモードで推定メモリが閾値を超える場合は実行前に警告する

## 6. 検証計画

- 平面基板（レンズなし）の透過率をフレネル係数の解析解と比較し、FDTD設定の妥当性を確認する
- 2Dと3Dで同一構造（1画素レンズ・垂直入射）の断面分布傾向が一致することを確認する
- メッシュ解像度を2倍にした際の集光効率変化が許容誤差内（目安±2%）であることを確認する

## 7. 技術選定の記録

- FDTDエンジン: **Meep**（無料・ローカル実行・実績十分）。
  Tidy3D は高速だが構造データを外部クラウドへ送信するためセキュリティルール上不採用。
  Lumerical は高額ライセンスのため不採用（導入する場合はエンジニア・購買相談）。
  MATLAB自作は開発量・検証リスクが過大なため不採用
- UI: ローカルWeb UI（FastAPI）。外部送信なし。将来のIIS社内公開（課題 I-005）にも接続しやすい
